title:搭建个人博客小结

这里是本人的个人小站（采用Hexo静态web框架并部署到github上），以后会随时更新一些本人学习过程中的一些日常总结，欢迎大家一起交流。

搭建过程

选择框架

当前流行的基于github Pages的个人站点有多种实现形式，下面简要介绍：

1.jekyll框架，也是github Pages的官方推荐web框架，运行环境基于ruby on rails,由于是github pages所以网上的资料相对较多，也更容易上手

2.pelican框架，是一个基于python的web静态框架，本人曾经尝试在windows环境下搭建过。需要基于makefile采用make命令编译；
安装也很方便，只需pip install pelican,pelican install markdown即可

3.octopress框架，值得注意的是同样基于jkelly生成，现在已经更新到3.0版本，值得去适用下；

4.Hexo框架，也即是本文需要着重介绍的静态web框架，基于node.js，相比以上三种框架，最大的特点是高效快速，这点本人有亲身体验。而且相对，修改，部署发布的命令都比较简单，只有常用的几个命令，下面介绍下具体的搭建过程；

1.安装node.js
直接到官网下载，安装即可，对于windows用户尤其方便，下载.exe或者.msi格式的安装文件，方便快捷

2.安装hexo，打开nodejs的command prompt采用命令npm install hexo-cli -g，安装完成后可以输入命令hexo version查看安装成功与否

3.使用hexo创建项目了
hexo init blog
cd blog
npm install
然后在本地开启一个测试hexo服务器，即hexo server，会出现如下提示：
[info] Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.
我们在留恋其中输入http://localhost:4000/,就可以看到生成的博客效果了，下面进行一些个人定制

仔细看下blog文件夹下的文件，我们可以发现主要有以下这些：

scaffolds 脚手架，也就是一个工具模板
scripts 写文件的js，扩展hexo的功能
source 存放博客正文内容
source/_drafts 草稿箱
source/_posts 文件箱,保存我们用于写博客的.md文件
themes 存放皮肤的目录
themes/landscape 默认的皮肤
_config.yml 全局的配置文件
db.json 静态常量

在这里，我们可以先定制_config.yml全局配置文件，值得注意的是初次进行修改，最好要先做一个备份，以免修改不当，或者误删了某些条目，切记。

站点信息: 定义标题，作者，语言
URL: URL访问路径
文件目录: 正文的存储目录
写博客配置：文章标题，文章类型，外部链接等
目录和标签：默认分类，分类图，标签图
归档设置：归档的类型
服务器设置：IP，访问端口，日志输出
时间和日期格式： 时间显示格式，日期显示格式
分页设置：每页显示数量
评论：外挂的Disqus评论系统
插件和皮肤：换皮肤，安装插件
Markdown语言：markdown的标准
CSS的stylus格式：是否允许压缩
部署配置：github发布

在修改配置文件时，尤其需要注意每个条目标题后都要留一个空格才能填写具体的内容，我就在这里浪费了不少弯路才发现问题；

4.将博客醒目部署到github上，前提是你的github庄户已经存在了username.github.io的repository，如果没有需要先创建。
采用hexo generate生成本地html，javascript，css文件，
注意还需要在__config.yml文件中，加入以下条目：

deplpy:
    type: git
    repository: https://github.com/zippera/zippera.github.io.git
    branch: master

修改完成后输入命令hexo deploy即可完成部署；

以上就完成了基本的博客搭建，由于缺乏个性化的定制，按上面步骤搭建出来的博客基本上是千篇一律的，想要你的博客与众不同还要进行主题，插件等设置。

tips：hexo常用的命令主要有以下四个

hexo g==hexo generate

hexo d==hexo deploy

hexo s==hexo server

hexo n==hexo new
